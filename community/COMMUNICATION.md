# 💬 Community Communication

## 📡 Channels
- Issue tracker: [github.com/Musawer1214/pashto-language-resources/issues](https://github.com/Musawer1214/pashto-language-resources/issues)
- Pull request review: [github.com/Musawer1214/pashto-language-resources/pulls](https://github.com/Musawer1214/pashto-language-resources/pulls)
- RFC or feature discussion: [Open a feature/RFC issue](https://github.com/Musawer1214/pashto-language-resources/issues/new?template=feature_request.md)
- Weekly async update thread: [Start a community update issue](https://github.com/Musawer1214/pashto-language-resources/issues/new?template=community_update.md)
- Monthly community review agenda: [Open a community review issue](https://github.com/Musawer1214/pashto-language-resources/issues/new?template=community_review.md)
- Release planning: [Open a release task](https://github.com/Musawer1214/pashto-language-resources/issues/new?template=release_task.md)
- Security reporting: [Private vulnerability report](https://github.com/Musawer1214/pashto-language-resources/security/advisories/new)

## 🗓️ Meeting rhythm
- Weekly async update issue for public progress notes and blockers
- Monthly community review issue for agenda, notes, and follow-ups

## 📏 Rules
- Keep technical decisions in public threads
- Summarize outcomes after meetings
- Tag maintainers only when blocked
- If any coordination happens outside GitHub, copy the final decision back into a public issue or PR
